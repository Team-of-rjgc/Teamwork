编码规范
type类型:
feat: 新增功能
fix: 修复bug
docs: 仅文档更改
style: 不影响代码含义的更改(空白、格式设置、缺失 分号等)
perf: 改进性能的代码更改
test:添加缺少的测试或更正现有测试
chore: 对构建过程或辅助工具和库(如文档)的更改
delete:删除功能或文件
modify:修改功能
代码：
常量大写，变量小写
关键、难懂的代码需要必要的注释，不要出现注释的代码
函数、变量、接口命名使用驼峰命名法，名字一定要有意义
常量一定要命名
重复的部分提取到函数中，避免代码冗长
格式要规范：括号要对齐，每行代码的长度不要超过80个字符，代码的缩进是4个空格
